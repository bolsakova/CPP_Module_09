# CPP_Module_09 - STL

## Brief Overview

Module_09 посвящен работе со Standard Template Library (STL) и алгоритмам обработки данных.

Если в предыдущих модулях основной акцент делался на Object-Oriented Programming, то здесь внимание смещается на выбор подходящих containers и эффективность алгоритмов.

В ходе выполнения модуля будут использоваться различные STL containers:
  - `std::map`
  - `std::stack`
  - `std::vector`
  - `std::deque`

Каждое упражнение посвящено отдельным структурам данных и алгоритмам.

|  Ex  |  Основная тема  
|------|-----------------
| ex00 | associative containers (`std::map`)
| ex01 | stack-based вычисления
| ex02 | Ford-Johnson merge-insertion sort

Главная цель модуля - научиться выбирать подходящий container для конкретной задачи и понимать, как это влияет на производительноть программы.

## STL Containers

STL (Standard Template Library) предоставляет готовые структуры данных и алгоритмы для работы с ними.

Containers можно условно разделить на несколько категорий:

### Sequential Containers

Элементы располагаются последовательно.

Наиболее распространённые:
  - `std::vector`
  - `std::deque`
  - `std::list`

### Associative Containers
 
Associative containers хранят данные в формате **key-value pair** и автоматически поддерживает элементы в отсортированном порядке.

Примеры:
  - `std::map`
  - `std::set`

### Container Adapters

Container Adapters построены поверх других containers и предоставляют ограниченный интерфейс.

К ним относятся:
  - `std::stack`
  - `std::queue`
  - `std::priority_queue`

## Почему STL важен?

STL позволяет не реализовывать структуры данных вручную, а использовать готовые и хорошо оптимизированные решения.

Правильный выбор container напрямую влияет на:
  - скорость поиска
  - скорость вставки элементов
  - объём используемой памяти
  - общую производительность программы

## Iterators

Iterator - это объект, позволяющий перемещаться по элементам container.

По своему поведению iterator похож на указатель.

Например:
  ```cpp
	  std::map<std::string, double>::iterator it;
	  // переход к следующему элементу
	  ++it;
	  // доступ к данным и возвращает key
	  it->first;
	  // возвращает value
	  it->second;
  ```

Таким образом, для `std::map<std::string, double>`:
  - `first` содержит дату
  - `second` содержит exchange rate

## std::pair

Элемент внутри `std::map` представляет собой объект типа:
  ```cpp
    std::pair<const Key, Value>
  ```

Например:
  ```cpp
	  std::pair<const std::string, double>
  ```

Состоит из двух частей:
  ```cpp
	  first	// key
	  second	// value
  ```

Для Bitcoin Exchange это соответствует:
	date -> exchange rate

## File Streams

Для чтения файлов используется класс - `std::ifstream`.
Например:
  ```cpp
	  std::ifstream file("data.csv");
  ```

Построчное чтение выполняется с помощью `std::getline()`.
Например:
  ```cpp
	  std::string line;
	  while (std::getline(file, line)) {}
  ```

Таким образом можно обрабатывать файл строка за строкой.

## String Parsing

Для разбиения строки удобно использовать - `std::stringstream`.

Например, строка:
  2011-01-3,0.3
может быть разделена на:
  date = "2011-01-03";
  price = 0.3;

Это позволяет преобразовывать текстовые данные в удобный для программы вид.

## Complexity

Различные containers имеют различную вычислительную сложность.
  - `std::vector` - поиск элемента: O(n)
  - `std::map` - поиск элемента: O(log n)
  - `std::stack` - добавление элемента: O(1), извлечение элемента: O(1)

Поэтому выбор container оказывает существенное влияние на производительность программы.
 
## Lexicographical comparison

Строки в С++ сравниваются посимвольно.
Например:
  ```cpp
	  "abc" < "abd" // вернет true, т.к. символ "с" меньше "d"
  ```

Формат даты YYYY-MM-DD обладает важным свойством:
  - лексикографический порядок совпадает с хронологическим порядком

Например:
  "2011-01-03" < "2012-05-01" и "2011-01-03" < "2011-02-10"
  => вернут `true`

Благодаря этому даты можно хранить как обычные `std::string`, не преобразуя их в специальные классы или timestamps.

Именно это делает `std::map<std::string, double>` идеальным решением для ex00.

## lower_bound()

Метод:
  ```cpp
	  container.lower_bound(key)
  ```
Возвращает итератор на первый элемент, к-рый >= key.

Например, если map содержит:
  2011-01-01
  2011-01-03
  2011-01-08
  2011-01-15
и выполнить `lower_bound("2011-01-05")`, то итератор будет указывать на "2011-01-08" поскольку это первая дата, не меньшая заданной.

Однако subject требует использовать ближайшую меньшую дату.
Поэтому необходимо перейти на предыдуший элемент:
  --it;
и получить "2011-01-03".

Именно этот exchange rate должен использоваться в вычислениях.

## LIFO Principle

`std::stack` работает по принципу - Last In First Out.
Последний добавленный элемент извлекается первым.

Пример:
  ```cpp
	  push(5)
	  push(7)
	  push(10)
  ```

Состояние stack - 10, 7, 5.
При последовательных вызовах `pop()` элементы будут извлекаться в порядке - 10, 7, 5.
Именно этот принцип используется при вычислении Reverse Polish Notation.

## Divide and Conquer

Многие алгоритмы сортировки используют подход - Divide and Conquer - к-рый состоит из 3 этапов:
  1. Divide	- разбить задачу на более мелкие
  2. Conquer	- решить каждую маленькую задачу
  3. Combine	- объединить результаты

Именно по этому принципу работает Merge Sort и Ford-Johnson algorithm.

## Binary Search

Binary search позволяет искать элемент не последовательно, а постоянно деля диапазон пополам.

Например:
  1 3 5 7 9 11 13

Поиск числа 9.
Проверяется середина: 7.
Поскольку 9 > 7 левая половина отбрасывается.
Далее рассматривается: 9 11 13.
После нескольких шагов элемент находится.

Сложность алгоритма - `O(log n)`, что значительно быстрее линейного поиска `O(n)`.

## Merge Sort

Merge Sort является алгоритмом сортировки с временной сложностью - `O(n log n)`.

Алгоритм состоит из двух этапов:

### Divide

Разделение массива:
  `8` `5` `3` `9`
  ↓
  `8` `5`
  `3` `9`
	↓
  `8`
  `5`
  `3`
  `9`

### Merge

Объединение отсортированных частей:
  `8`
  `5`
  ↓
  `5` `8` 
и
  `3`
  `9`
	↓
  `3` `9`
после чего
  `5` `8`
  `3` `9`
объединяются в
  `3` `5` `8` `9`

## Container Adapters

Некоторые STL containers построены поверх других containers.
 
Например: 
  `std::stack` по умолчанию использует `std::deque` и предоставляет ограниченный интерфейс:
  ```cpp
	  push()
	  pop()
	  top()
	  empty()
	  size()
  ```

Пользователь не может обращаться к произвольному элементу stack. Это помогает сохранять принцип LIFO.

# ex00 - Bitcoin Exchange

## Subject Overview

Необходимо написать программу `btc`, к-рая вычисляет стоимость некоторого кол-ва биткоинов на указанную дату.

Программа использует два источника данных:

### database

Файл `data.csv` содержит исторические данные курса Bitcoin.
Например:
  date,exchange_rate
	2011-01-03,0.3
	2011-01-08,0.32
	2011-01-11,7.1

### input file

Второй файл передаётся через command line argument:
 ```bash
	./btc input.txt
 ```
и содержит строки вида:
  date | value

Например:
  2011-01-03 | 3
  2011-01-08 | 1
  2011-01-11 | 5

Для каждой строки необходимо:
  1. прочитать дату
  2. прочитать кол-во биткоинов
  3. найти exchange rate
  4. вычислить `value * exchange rate`
  5. вывести результат

## Main Difficulty

Основная сложность задания заключается в том, что дата из input file может отсутствовать в database.
Например:
  Database:
  2011-01-03
  2011-01-08
  2011-01-11

  Input:
  2011-01-05

Такой даты нет в database.
Согласно subject необходимо использовать ближайшую меньшую дату, а не следующую. Именно поэтому для данного задания идеально подходит `std::map`.

## Why `std::map`?

Необходимо хранить соответствие:
  date -> exchange rate

Такую структуру удобно представить как:
```cpp
	std::map<std::string, double>
```

где `key`    - дата 2011-01-03
    `value`  - курс Bitcoin 0.3

Например:
```cpp
  {
      "2011-01-03" → 0.3
      "2011-01-08" → 0.32
      "2012-01-11" → 7.1
  }
```

Преимущества `std::map`:

### automatic sorting

Эл-ты автоматически поддерживаются в отсортированном порядке.

### logarithmic search

Поиск выполняется за `O(log n)`.

### lower_bound()

Позволяет быстро найти ближайшую дату.

## Internal Structure of `std::map`

`std::map` реализован на основе self-balancing binary search tree.
В большинстве реализаций используется Red-Black Tree. Благодаря этому:
  - insertion выполняется за `O(log n)`
  - search выполняется за `O(log n)`
  - эл-ты всегда остаются отсортированными

Поэтому не требуется дополнительно сортировать database после загрузки.

## Why strings are enough for dates?

Даты имеют формат YYYY-MM-DD.
Такой формат обладает важным свойством - лексикографическое сравнение полностью совпадает с хронологическим, поэтому нет необходимости преобразовывать даты в timestamp или использовать специальные классы.

# ex01 - Reverse Polish Notation

## Subject Overview

Необходимо реализовать программу `RPN`, к-рая вычисляет математическое выражение, записанное в формате **Reverse Polish Notation (RPN)**.

Выражение передаётся программе через command-line argument.

Например:
```bash
  ./RPN "7 7 * 7 -"
```

Результатом работы программы будет: 42

в отличии от обычной математической записи, в Reverse Polish Notation отсутствуют скобки и приоритет операций. Порядок вычислений полностью определяется последовательностью токенов.

## What is Reverse Polish Notation?

Reverse Polish Notation (RPN), также известная как **postfix notation**, - это способ записи математическиъ выражений, при к-ром оператор записывается после операндов.

Например:
  - Обычная запись: 3 + 4
  - RPN: 3 4 +

Более сложный пример: (3 + 4) * 5
В RPN выглядит так: 3 4 + 5 *

Главное преимущество RPN заключается в том, что для вычисления выражения не требуются скобки и правила приоритета операций.

## Why `std::stack`?

Для вычисления выражения необходимо постоянно работать с последними добавленными числами.

Именно поэтому наиболее подходящим контейнером является:
`std::stack<int>`

`std::stack` работает по принципу LIFO (Last In First Out).
Последний добавленный элемент извлекается первым.

Именно такое поведение идеально подходит для вычисления RPN.

## Algorithm

Алгоритм вычисления выражения состоит из следующих шагов:

1. Читать выражение слева направо.
2. Если встретить число - поместить его в stack.
3. Если встретился оператор:
  - извлечь два верхних элемента stack
  - выполнить оерацию
  - поместить результат обратно в stack
4. осле обработки всех токенов в stack должен остаться ровно один элемент - результат вычисления.

## Example

Выражение: 7 7 * 7 -

Пошаговое выполнение :
  Token: 7
  Stack: 7

  Token: 7
  Stack: 7
         7

  Token: *
  7 * 7 = 49
  Stack: 49

  Token: 7
  Stack: 49
         7

  Token: -
  49 - 7 = 42
  Stack: 42

После обработки всех токенов в stack остался один элемент: 42
Именно он является результатом вычисления.

## Input Validation

Программа должна обрабатывать только корректные выражения.
Необходимо проверить:
  - выражение не пустое
  - используются только допустимые операторы "+ - * /"
  - операнды являются числами
  - перед выполнением операции в stack находятся минимум два элемента
  - после окончания вычислений в stack остаётся ровно один элемент

Во всех остальных случаях программа должна вывести: Error

## Design

Основной контейнер задания: `std::stack<int>`

Класс `RPN` отвечает за:
  - разбор выражения
  - определение типа токена
  - выполнение математических операций
  - управление stack
  - возврат результата вычислений

## Why are the last two elements used?

При встрече оператора программа должна выполнить операцию над двумя последними числами, к-рые были помещены в `std::stack`.

Например, выражение "7 3 -" обрабатывается следующим образом:
  Token: 7
  Stack: 7

  Token: 3
  Stack: 7
         3
  
Встретили оператор: -

Из stack извлекаются два последних элемента. Однако важно помнить порядок извлечения. Первым извлекается верхний элемент stack:
```cpp
  a = stack.top();
  stack.pop();
```
Получаем: а = 3

Затем извлекается следующий элемент:
```cpp
  b = stack.top();
  stack.pop();
```
Получаем: b = 7

Теперь выполняется операция: "b - a",
т.е. "7 - 3 = 4", а не "3 - 7 = -4"

Полученный результат снова помещается в stack.
  Stack: 4

Поэтому при выполнении операций всегда используется порядок:
  b op a

Это особенно важно для операций "/ -".
Для "+ *" результат совпадет независимо от порядка операндов, однако для единообразия алгоритма всегда используется схема:
  a = верхний элемент stack
  b = следующий элемент stack
  result = b op a

# ex02 - PmergeMe

## Subject Overview

Необходимо написать программу `PmergeMe`, к-рая принимает последовательность положительных integer numbers через command-line arguments и сортирует их с помощью Ford-Johnson algorithm, также известного как merge-insertion sort.

Программа должна:
  - проверить, что все arguments являются положительными integer numbers
  - сохранить исходную последовательность
  - отсортировать ее двумя разными STL containers
  - вывести sequence до сортировки и после сортировки
  - измерить время работы алгоритма для каждого container

В этом задании необходимо использовать два разных containers.

В моём решении используются:
  - `std::vector`
  - `std::deque`

## Main idea of Ford-Johnson algorithm

Ford-Johnson algorithm старается уменьшить кол-во comparisons во время сортировки.

Основная идея:
1. разбить эл-ты на pairs
2. в каждой pair определить bigger и smaller element
3. рекурсивно отсортировать bigger elements
4. построить main chain
5. вставить smaller elements в main chain с помощью binary search
6. использовать Jacobsthal order, чтобы выбрать более выгодный порядок вставки

## Why two containers?

Subject требует использовать два разных containers.
Алгоритм реализуется отдельно для:
  `std::vector<int>` и `std::deque<int>`

Это позволяет сравнить, как один и тот же algorithm работает с разными data structures.

## Program output

Программа выводит:

  Before: 3 5 9 7 4
  After: 3 4 5 7 9
  Time to process a range of 5 elements with std::vector : ...
  Time to process a range of 5 elements with std::deque : ...

## Error handling

Программа должна вывести - Error, если
  - argument не является числом
  - число отрицательное
  - число равно нулю
  - число выходит за пределы int
  - отсутствуют arguments

## Why are two containers used?

По subject необходимо реализовать один и тот же алгоритм для двух разных STL контейнеров  сравнить их время выполнения.
В данном задании используются:
  `std::vector<int>` и `std::deque<int>`

Несмотря на то, что алгоритм сортировки полностью одинаковый, внутренне устройство контейнеров отличается, поэтому производительность также может отличаться.

### `std::vector`

`std::vector` хранит эл-ты в одной непрерывной области памяти (contiguous memory).

Преимущества:
  - быстрый random access (`O(1)`)
  - хорошая cache locality
  - обычно показывает лучшую производительность при последовательном обходе

Недостатки:
  - вставка эл-та в середину требует сдвига всех последующих эл-тов

### `std::deque`

`std::deque` состоит из нескольких небольших непрерывных блоков памяти.

Преимущества:
  - эффективное добавление эл-тов в начало и конец
  - не требует полного перераспределения памяти при увеличении размера

Недостатки:
  - память не является полностью непрерывной
  - cache locality хуже, чем у `std::vector`
  - random access немного менее эффективен

### Why compare them?

Ford-Johnson algorithm актично выполняет
  - сравнение эл-тов
  - бинарный поиск
  - вставку эл-тов в середину последовательности

Поэтому интересно посмотреть, как один и тот же алгоритм ведёт себя на containers с разной организацией памяти.

Для этого программа измеряет время выполнения отдельно для `std::vector` и `std::deque`.

## Note

В данном задании измеряется не скорость работы контейнера, а скорость работы алгоритма при использовании разных контейнеров.

Алгоритм остаётся полностью одинаковым, меняется только структура данных, в к-рой он выполняется.
