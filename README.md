## STL Containers

 STL (Standard Template Library) предоставляет готовые структуры данных и алгоритмы для работы с ними.

 Containers можно условно разделить на несколько категорий:

### Sequential Containers

 Элементы располагаются последовательно.

 Наиболее распространённые:
   - `std::vector`
   - `std::deque`
   - `std::list`

### Associative Containers
 
 Associative containers хранят данные в формате **key-value pair** и автоматически поддерживает элементы в отсортированном порядке.

 Примеры:
   - `std::map`
   - `std::set`

### Container Adapters

 Container Adapters построены поверх других containers и предоставляют ограниченный интерфейс.

 К ним относятся:
   - `std::stack`
   - `std::queue`
   - `std::priority_queue`

## Почему STL важен?

 STL позволяет не реализовывать структуры данных вручную, а использовать готовые и хорошо оптимизированные решения.

 Правильный выбор container напрямую влияет на:
   - скорость поиска
   - скорость вставки элементов
   - объём используемой памяти
   - общую производительность программы

## Iterators

 Iterator - это объект, позволяющий перемещаться по элементам container.

 По своему поведению iterator похож на указатель.

 Например:
 ```cpp
	std::map<std::string, double>::iterator it;
	// переход к следующему элементу
	++it;
	// доступ к данным и возвращает key
	it->first;
	// возвращает value
	it->second;
 ```

 Таким образом, для `std::map<std::string, double>`:
   - `first` содержит дату
   - `second` содержит exchange rate

## std::pair

 Элемент внутри `std::map` представляет собой объект типа:
 ```cpp
	std::pair<const Key, Value>
 ```

 Например:
 ```cpp
	std::pair<const std::string, double>
 ```

 Состоит из двух частей:
 ```cpp
	first	// key
	second	// value
 ```

 Для Bitcoin Exchange это соответствует:
	date -> exchange rate

## File Streams

 Для чтения файлов используется класс - `std::ifstream`.
 Например:
 ```cpp
	std::ifstream file("data.csv");
 ```

 Построчное чтение выполняется с помощью `std::getline()`.
 Например:
 ```cpp
	std::string line;
	while (std::getline(file, line)) {}
 ```

 Таким образом можно обрабатывать файл строка за строкой.

## String Parsing

 Для разбиения строки удобно использовать - `std::stringstream`.

 Например, строка:
   2011-01-3,0.3
 может быть разделена на:
   date = "2011-01-03";
   price = 0.3;

 Это позволяет преобразовывать текстовые данные в удобный для программы вид.

## Complexity

 Различные containers имеют различную вычислительную сложность.

 - `std::vector` - поиск элемента: O(n)
 - `std::map` - поиск элемента: O(log n)
 - `std::stack` - добавление элемента: O(1), извлечение элемента: O(1)

 Поэтому выбор container оказывает существенное влияние на производительность программы.
 
## Lexicographical comparison

 Строки в С++ сравниваются посимвольно.
 Например:
 ```cpp
	"abc" < "abd" // вернет true, т.к. символ "с" меньше "d"
 ```

 Формат даты YYYY-MM-DD обладает важным свойством:
   - лексикографический порядок совпадает с хронологическим порядком

 Например:
   "2011-01-03" < "2012-05-01" и "2011-01-03" < "2011-02-10"
   => вернут `true`

 Благодаря этому даты можно хранить как обычные `std::string`, не преобразуя их в специальные классы или timestamps.

 Именно это делает `std::map<std::string, double>` идеальным решением для ex00.

## lower_bound()

 Метод:
 ```cpp
	container.lower_bound(key)
 ```
 Возвращает итератор на первый элемент, к-рый >= key.

 Например, если map содержит:
  2011-01-01
  2011-01-03
  2011-01-08
  2011-01-15
 и выполнить `lower_bound("2011-01-05")`, то итератор будет указывать на "2011-01-08" поскольку это первая дата, не меньшая заданной.

 Однако subject требует использовать ближайшую меньшую дату.
 Поэтому необходимо перейти на предыдуший элемент:
   --it;
 и получить "2011-01-03".

 Именно этот exchange rate должен использоваться в вычислениях.

## LIFO Principle

 `std::stack` работает по принципу - Last In First Out.
 Последний добавленный элемент извлекается первым.

 Пример:
 ```cpp
	push(5)
	push(7)
	push(10)
 ```

 Состояние stack - 10, 7, 5.
 При последовательных вызовах `pop()` элементы будут извлекаться в порядке - 10, 7, 5.
 Именно этот принцип используется при вычислении Reverse Polish Notation.

## Divide and Conquer

 Многие алгоритмы сортировки используют подход - Divide and Conquer - к-рый состоит из 3 этапов:
   1. Divide	- разбить задачу на более мелкие
   2. Conquer	- решить каждую маленькую задачу
   3. Combine	- объединить результаты

 Именно по этому принципу работает Merge Sort и Ford-Johnson algorithm.

## Binary Search

 Binary search позволяет искать элемент не последовательно, а постоянно деля диапазон пополам.

 Например:
   1 3 5 7 9 11 13

 Поиск числа 9.
 Проверяется середина: 7.
 Поскольку 9 > 7 левая половина отбрасывается.
 Далее рассматривается: 9 11 13.
 После нескольких шагов элемент находится.

 Сложность алгоритма - `O(log n)`, что значительно быстрее линейного поиска `O(n)`.

## Merge Sort

 Merge Sort является алгоритмом сортировки с временной сложностью - `O(n log n)`.

 Алгоритм состоит из двух этапов:

### Divide

 Разделение массива:
  `8` `5` `3` `9`
	↓
  `8` `5`
  `3` `9`
	↓
  `8`
  `5`
  `3`
  `9`

### Merge

 Объединение отсортированных частей:
  `8`
  `5`
    ↓
  `5` `8` 
	и
  `3`
  `9`
	↓
  `3` `9`
	после чего
  `5` `8`
  `3` `9`
	объединяются в
  `3` `5` `8` `9`

## Container Adapters

 Некоторые STL containers построены поверх других containers.
 
 Например: 
 `std::stack` по умолчанию использует `std::deque` и предоставляет ограниченный интерфейс:
 ```cpp
	push()
	pop()
	top()
	empty()
	size()
 ```

 Пользователь не может обращаться к произвольному элементу stack. Это помогает сохранять принцип LIFO.

# ex00 - Bitcoin Exchange

## Subject Overview

 Необходимо написать программу `btc`, к-рая вычисляет стоимость некоторого кол-ва биткоинов на указанную дату.

 Программа использует два источника данных:

 Например:
	date,exchange_rate
	2011-01-03,0.3

 Второй файл передаётся через command line argument:
 ```bash
	./btc input.txt
 ```
